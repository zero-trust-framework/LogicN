# Language Positioning Principles

LogicN should describe itself by its own design goals, not by ranking itself
against other language communities.

## Product Direction

LogicN is a beta language/runtime concept focused on:

```text
secure web runtime behavior
strict types
memory-safe values
typed API contracts
explicit effects
package authority
secret-safe output
source maps
machine-readable reports
AI-safe project context
future compute target planning
```

## Positioning Rules

Use positive, direct wording:

```text
LogicN makes security boundaries visible before code runs.
LogicN makes APIs typed and reportable.
LogicN treats package authority as explicit.
LogicN keeps AI-readable project context redacted and source-mapped.
LogicN starts with a secure web runtime before advanced output targets.
```

Avoid ranking language communities or implying replacement of their work.

## What LogicN Should Learn

LogicN should keep the best general lessons from mature ecosystems:

| Lesson | LogicN direction |
|---|---|
| Readability matters | Keep syntax regular, direct and teachable. |
| Strong types help teams | Make public boundaries explicit and checked. |
| Memory behavior matters | Provide ownership, views, resource scopes and reports. |
| Package tooling matters | Lock dependencies, declare authority and report package effects. |
| Developer workflow matters | Provide `check`, `run`, `serve`, `test` and reports. |
| Runtime performance matters | Cache typed IR, route manifests, schemas and policy decisions. |
| AI tooling needs facts | Generate project graphs, source maps and redacted summaries. |

## Current Claim

```text
LogicN is a security-first, memory-safe, AI-readable language/runtime direction
for typed web applications, APIs and agent workflows.
```

## Future Claim

After the parser, checker, runtime, memory model, package manager and report
pipeline mature, LogicN can make stronger claims about production readiness.

Until then, the correct framing is:

```text
prototype tooling
planning documents
checked examples
clear roadmap
secure runtime first
```
