# Glossary

```text
binary logic      = 2-state logic
ternary logic     = 3-state logic
quaternary logic  = 4-state logic
n-state logic     = configurable future logic width
omni logic        = abstraction over multiple logic widths
logic width       = number of available logic states
logic target      = compiler/runtime target with declared logic capability
memory pressure   = low-memory condition before hard failure
disk spill        = moving approved temporary data from memory to disk
checkpoint        = saved safe state for rollback
diagnostic        = structured warning, error, fatal or info report
target fallback   = using another backend when the preferred target is unavailable
flow             = LogicN's function-like unit of behaviour, checked for inputs, outputs, effects, security, reports, rollback, AI context and target optimisation
secure flow      = flow with security-sensitive behaviour and explicit checks/effects
pure flow        = deterministic flow with no side effects
```
