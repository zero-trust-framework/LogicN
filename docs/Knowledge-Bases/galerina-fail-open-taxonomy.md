## 2. Component-Health Dev-Tool Spec (owner's explicit ask)

Owner asked to "check each component (runtime, compiler, etc.) — builds / working / tests". A rich quality layer already exists but **every piece is repo-GLOBAL — none pivots to a per-component view.** The one missing capability is the per-component PIVOT, and it is exactly the lens that surfaces today's failures at a glance (a `security`/`compiler` row with `failopen>0`; a native-bridge `dangcov` survivor).

**Net-new tool:** `scripts/component-health.mjs` (confirmed **absent** on disk) — `node scripts/component-health.mjs [--json] [--component <role>] [--fast] [--full]`. A per-component roll-up that JOINS existing checks to `galerina.workspace.json` roles. It **re-implements no check.**

**Row model.** One row per component, where component = a `galerina.workspace.json` role (compiler/runtime/network/security/config/reports/projectGraph/tower-citizen/…, ~40 role keys over 90 packages) resolved to its package dir, plus synthetic `tower-citizen`, `core-network`, `tooling` (scripts/) rows. Unmapped packages roll up under family prefix (`galerina-data-*`, `galerina-web-*`, `galerina-target-*`) so all 90 packages are covered (universal-coverage rule — no orphan components).

**Columns** (each cell = status + number, from a named existing producer):
1. **builds?** ← per-pkg `npm run typecheck` (reuse `run-all-tests.cjs` dist-smart dispatch) → ok/FAIL
2. **tests** ← `run-all-tests.cjs` named-package mode → pass/fail + count (reuse `parseCounts`)
3. **failopen lints** ← Lane-A class detectors filtered to the pkg dir: `lint-wat-inline-comments` (now landed), `lint-fungi`, value-state/tier coverage assertions → violations
4. **dangcov** (dangerous-path coverage) ← `audit-mutation` surviving-mutants + `audit-provenance`/security findings attributed to files under the pkg dir → killed/total
5. **prov fresh?** ← `audit-provenance --json` filtered to artifacts whose sources include the pkg → fresh/STALE/MISSING

**Reuse vs net-new.**
- **REUSE (do not fork/duplicate):** `run-all-tests.cjs` (named-pkg mode already returns per-pkg `{status,tests,code}`, `parseCounts`, dist-smart dispatch — import/spawn it); the 5 engines `audit-mutation`/`audit-provenance`/`audit-{diagnostic-codes,doc-drift}`/`lint-fungi`/`lint-wat-inline-comments` (all `--json` + `VIOLATIONS: N`); `lib/provenance.mjs` + the ARTIFACTS registry; `version.json` `testCountByPackage` (per-package — the `--fast` fallback); `galerina.workspace.json` role keys (the taxonomy — don't invent one); `run-phase-close.mjs` as the spawn-children-and-summarise precedent.
- **NET-NEW (~250–350 LOC, one file):** (1) role→pkgDir resolver from workspace.json + family-prefix fallback (all 90 map); (2) `componentOf(filePath)` longest-prefix bucketer — **the missing pivot** that routes repo-global findings into per-component cells (audits emit paths but never bucket them); (3) the 5-column row assembler + verdict; (4) table + `--json` + `build/component-health/*` writer with a `provenance.json` sidecar (so it audits clean under its OWN provenance column — dogfood).

**Verdict logic:** RED if build fails OR tests fail OR a fail-open lint >0 OR a dangerous-path mutant survives; AMBER if provenance STALE only; GREEN otherwise.

**Output shape.** DEFAULT human table (`component | builds | tests | failopen | dangcov | prov | verdict`), final `N/M components green` line, **exit code = #RED** (CI-gateable, mirrors `lint-conventions`). `--json` = `{generatedAt, provenance:{tool,gitCommit,builtAt}, components:[{role,pkg,builds,tests:{status,count},failOpenLints,dangerousCoverage:{killed,total},provenance,verdict}], summary:{green,amber,red,total}}`. Persist to `build/component-health/component-health.{json,md}` + `provenance.json`.

**Tie-in:** register as a CHECK in `run-phase-close.mjs` after `graph:reindex`. v2 only: annotate project-graph Package nodes with the verdict (not required for v1).

**Effort: SMALL–MEDIUM, ~0.5–1 day.** 2–3h resolver+bucketer+family fallback; 2–3h wire the 5 producers + verdict; 1–2h table/`--json`/writer/provenance/register/self-test (`scripts/tests/`, ~60 LOC). Cost driver is run time not code, so **`--fast` (read `version.json`) is the default; `--full` does the live per-pkg build+test pass** (multi-minute across 90 pkgs).

**Guardrails (recorded):** (a) keep it informational/exit=#RED like `lint-conventions` — NOT a hard pre-commit gate on day one (baselines are non-zero); (b) given today's #7 shared-tree collision, the tool must be **pure-read** of build artifacts + `version.json` in `--fast` mode and must NOT `git add`/commit anything — it reports, it does not mutate the tree.

**Sequencing dependency met:** `lint-wat-inline-comments.mjs` (the failopen-column detector) is **confirmed on disk** — the column can include the #163 class from v1.

## 4. New Rules · New Docs · Dangerous-Path Test Patterns

**New feedback rules** (`C:\Users\desig\.claude\projects\C--Users-desig\memory\`):
- **`feedback-verify-fail-closed-by-running.md`** — Verify a fail-closed claim by RUNNING the behaviour, not by reading the trap text. The #163 hub read `(unreachable)` and certified "already traps" without tracing that inline `;;` → wabt-reject → `valid:true` stub → wrong value `5`. A fail-closed assertion is verified only when you EXECUTE/TRACE the dangerous path end-to-end and observe the deny/trap. The gating condition often lives in a different file than the text you read. Applies to every "already enforced/traps/denies" claim before building on top. SEC-002 mutation is this discipline mechanized.
- **`feedback-fix-the-class-not-the-instance.md`** — When you fix one fail-open site, fix the CLASS: grep for siblings, then add a lint so it can't recur. The first #163 fix patched only the 8 `(unreachable)` traps and missed static-const `:1021` + bitfield `:1035` of the same shape — found only when the owner asked. A documented code-comment (`:1038-1039`) is NOT enforcement. Three mandatory steps: (1) fix the site, (2) characterize the class + grep every sibling, (3) add a mechanical gate (lint/test) and register it in `lint-conventions.mjs`.
- **`feedback-dangerous-branch-must-be-exercised.md`** — A fail-closed gate on a branch that never runs in CI is untested — mock/stub the precondition so the dangerous branch executes. The bitnet commit gate (CWE-862) was missing and tests never caught it because no native addon exists on a clean checkout. CI must inject a stub satisfying the precondition (native/key/prod-only/hardware), with assertions that the gate denies AND allows. Pair with SEC-002: re-introduce the hole on the native branch and assert the mock-native run is denied.
- **`feedback-tier-floor-not-just-effect-ceiling.md`** — Enforce a tier floor (declared tier ⊇ inferred-minimum tier), not just an effect ceiling. A `guarded flow` doing `http.post` skipped every secure-only obligation (GOV-010/006). Effect-subset is necessary but not sufficient; under-declaration of tier is privilege-escalation-by-omission. Net-new floor FUNGI-TIER-001. Audit every tiered declaration (flow kinds, policies, profiles) for a matching floor.
- **Reaffirm `feedback-fleet-shared-tree-pathspec.md`** (already recorded) — every fleet/worker session stages with explicit pathspec only (`git add packages-galerina/<own-pkg>`), never `git add -A`/`.`/`commit -a`; prefer per-session worktrees; don't rewrite shared-branch history while workers are active.
