# Package Declaration Syntax

## Definition

Galerina packages are declared, versioned, and governed. Every package must declare
its identity, version, dependencies, and permissions. The runtime verifies
packages against lockfiles, hashes, and registry signatures before linking.

## Package Families

| Prefix | Purpose |
| --- | --- |
| `galerina-core-*` | Required core packages (compiler, runtime, security, memory) |
| `galerina-ai-*` | AI and inference packages |
| `galerina-target-*` | Compile target backends (WASM, GPU, photonic) |
| `galerina-framework-*` | Application framework packages |
| `galerina-devtools-*` | Developer tooling |
| `galerina-tools-*` | CLI and build tools |

## Required Core Packages

```text
galerina-core             — language core
galerina-core-compiler    — compiler
galerina-core-runtime     — runtime engine
galerina-core-security    — security primitives
galerina-core-memory      — memory model
```

## Optional Official Packages

```text
galerina-core-worker      — worker pools
galerina-core-network     — network boundary
galerina-core-compute     — hardware compute targets
galerina-ai               — AI and ML integration
galerina-data             — data access
```

## Dependency Declaration

Declared in the project manifest (`.fungi` or separate manifest file):

```galerina
package my_app {
  version: "1.0.0"

  dependencies {
    galerina-core >= 0.1
    galerina-core-runtime >= 0.1
    galerina-ai >= 0.1
  }
}
```

## Permission Declaration

Every package declares the permissions it needs:

```galerina
package galerina-ai {
  version: "0.1"

  permissions {
    require network
    require execute
  }

  dependencies {
    galerina-core >= 0.1
  }
}
```

## Lockfile

The lockfile (`galerina.lock`) records the exact resolved versions, hashes,
licences, and permissions for every dependency:

```text
package name
version
hash
licence
permissions
target compatibility
```

The lockfile is generated by the package resolver and must not be edited by hand.

## Package Resolver Governance

Before any package is linked into the runtime:

```text
1. verify package identity
2. check lockfile hash
3. verify registry signature
4. check capabilities and permissions
5. check effects declaration
6. check licence and policy
7. check trust status
8. verify dependencies
9. check for conflicts
10. link approved modules into Governed IR with provenance report
```

Packages that fail verification are rejected. The runtime does not execute
unverified packages.

## Module Visibility

Modules within a package declare their visibility:

```galerina
module orders {
  expose: [create_order, get_order, cancel_order]
  internal: [validate_stock, compute_total]
}
```

Unexposed functions are not accessible to other packages.

## Security Rules

```text
Packages are untrusted by default until verified.
Imports are not trust — they require resolver verification.
Dependencies cannot exceed the permissions granted to the parent package.
The resolver checks supply-chain signatures and registry certification.
Dynamic loading still goes through Authority Control and resolver policy.
```

## CI Security Checks

The build system should fail CI if:

```text
unsigned packages are detected
dependencies request risky permissions without lockfile acknowledgement
lockfile is missing or stale
package hash does not match registry
```

## Core Principle

```text
Every package is declared, verified, and governed.
The resolver finds packages. The runtime links them.
Neither trusts the other without cryptographic proof.
```
