# Runtime Audit Log Format

## Status

```text
Package: galerina-core-reports
Area: runtime audit logs, execution proof, denials, evidence
Version target: v1 (canonical — spore.runtime.audit.v1)
Format: JSONL, one RuntimeAuditEvent per line
Implementation status: fully specified, not yet finalised in package
Canonical diagnostic ranges:
  - SPORE-AUDIT-001 through SPORE-AUDIT-005
  - SPORE-REPORT-001 through SPORE-REPORT-005
  - SPORE-PROOF-001 through SPORE-PROOF-005
  - SPORE-DENIAL-001 through SPORE-DENIAL-005
  - SPORE-EVIDENCE-001 through SPORE-EVIDENCE-005
```

Runtime audit logs are security-relevant records emitted by the Galerina runtime and related governance tooling.

They are distinct from normal application logs.

Audit logs answer:

```text
what happened
when it happened
which artefact performed it
which authority was requested
which policy allowed or denied it
which evidence proves the decision
```

---

## Design Goals

```text
JSONL output
append-only events
deterministic schema
machine-readable evidence
tamper-evident proof hashes
secret-safe report values
clear denial reports
runtime/CLI/deploy compatibility
```

Audit logs must never contain raw secrets, bearer tokens, private keys, passwords, webhook secrets, session cookies, or unredacted protected data.

---

## JSONL File Format

The runtime audit log is a JSON Lines file.

```text
runtime-audit.jsonl
```

Each line is one complete JSON object:

```jsonl
{"schemaVersion":"spore.runtime.audit.v1","eventId":"evt_001","status":"allowed"}
{"schemaVersion":"spore.runtime.audit.v1","eventId":"evt_002","status":"denied"}
```

Rules:

```text
one event per line
no trailing commas
UTF-8 encoded
append-only
newline terminated
stable field names
no raw secrets
```

## RuntimeAuditEvent v0.2

```ts
export interface RuntimeAuditEvent {
  schemaVersion: "spore.runtime.audit.v1"

  /** Stable event id generated by the runtime or CLI. */
  eventId: string

  /** Correlates all events for one execution attempt. */
  executionId: string

  /** Optional distributed trace id. */
  traceId?: string

  /** Optional span id for nested runtime work. */
  spanId?: string

  /** ISO timestamp. */
  timestamp: string

  /** High-level governance category for this event. */
  category:
    | "runtime"
    | "network"
    | "capability"
    | "effect"
    | "secret"
    | "deployment"
    | "webhook"
    | "verification"

  /** Runtime status for this audit event. */
  status: RuntimeAuditStatus

  /** Human-readable summary of what happened or was denied. */
  message: string

  /** Machine-readable event type. */
  eventType:
    | "execution.started"
    | "capability.requested"
    | "capability.allowed"
    | "capability.denied"
    | "effect.performed"
    | "boundary.crossed"
    | "boundary.denied"
    | "manifest.verified"
    | "deployment.denied"
    | "execution.completed"
    | "execution.failed"

  /** Build/deploy artefact that produced the event. */
  artefact: AuditArtefactReference

  /** Actor that attempted the operation. */
  actor: AuditActor

  /** Operation that happened or was attempted. */
  action: AuditAction

  /** Policy decision metadata. */
  policy?: AuditPolicyDecision

  /** Execution proof hashes associated with this event. */
  proof?: ExecutionProofHashes

  /** Denial report when status is denied or failed. */
  denial?: DenialReport

  /** Evidence entries supporting the event. */
  evidence: RuntimeEvidence[]

  /** Secret-safe metadata only. */
  metadata: Record<string, AuditReportValue>
}
```

## ExecutionProofHashes

Execution proof uses five hashes — field names confirmed canonical.

```ts
export interface ExecutionProofHashes {
  /** sha256 of the build manifest file. */
  manifestSha256: string

  /** sha256 of the runtime audit JSONL file. */
  auditSha256: string

  /** sha256 of the evidence report file. */
  evidenceSha256: string

  /** sha256 of the denial report file. */
  denialSha256: string

  /** sha256 of the build artefact file. */

}
```

The five hashes are used by:

```text
galerina verify
galerina deploy
galerina explain
runtime audit reports
execution proof reports
```

## ExecutionProof v1 — canonical fixed shape

v1 is intentionally simple and fixed to the five confirmed hashes.

```ts
export interface ExecutionProofV1 {
  schemaVersion: "spore.execution.proof.v1"

  /** Stable proof identifier. */
  proofId: string

  /** ISO timestamp when this proof was generated. */
  generatedAt: string

  /** Five canonical file-content hashes. */
  hashes: ExecutionProofHashes
}
```

## ExecutionProof v2 — extensible sections

v2 preserves the v1 `hashes` field for compatibility and adds optional extensible sections.

```ts
export interface ExecutionProofV2 {
  schemaVersion: "spore.execution.proof.v2"

  proofId: string

  generatedAt: string

  /**
   * Retain v1 fields for compatibility.
   */
  hashes: ExecutionProofHashes

  /**
   * Optional extensible sections for future proof material.
   */
  sections?: ExecutionProofSection[]

  /**
   * Optional chain/reference metadata.
   */
  references?: ExecutionProofReference[]
}

export interface ExecutionProofSection {
  name: string
  algorithm: "sha256" | "sha384" | "sha512"
  hash: string
  path?: string
  contentType?: string
}
```

## Upgrade: v1 → v2

```ts
export function upgradeExecutionProofV1ToV2(
  proof: ExecutionProofV1
): ExecutionProofV2 {
  return {
    schemaVersion: "spore.execution.proof.v2",
    proofId: proof.proofId,
    generatedAt: proof.generatedAt,
    hashes: proof.hashes,
    sections: [
      { name: "manifest",  algorithm: "sha256", hash: proof.hashes.manifestSha256 },
      { name: "audit",     algorithm: "sha256", hash: proof.hashes.auditSha256 },
      { name: "evidence",  algorithm: "sha256", hash: proof.hashes.evidenceSha256 },
      { name: "denial",    algorithm: "sha256", hash: proof.hashes.denialSha256 },
      { name: "artefact",  algorithm: "sha256", hash: proof.hashes.artefactSha256 },
    ],
  }
}
```

```text
v1 ExecutionProof is intentionally fixed to five hashes.
v2 adds extensible sections but preserves the v1 hashes field for compatibility.
```

## JSONL Writer Contract

```ts
export interface JsonlAuditWriter {
  append(event: RuntimeAuditEvent): Promise<void>
  flush?(): Promise<void>
  close?(): Promise<void>
}
```

Contract rules:

```text
1. Writer appends only.
2. Each event serializes to exactly one JSON line.
3. Each line ends with "\n".
4. Writer must reject invalid schemaVersion.
5. Writer must reject unsafe raw secrets.
6. Writer must not pretty-print JSONL.
7. Writer must preserve event order per writer instance.
```

Serialisation helper:

```ts
export function serializeAuditEvent(
  event: RuntimeAuditEvent
): string {
  if (event.schemaVersion !== "spore.runtime.audit.v1") {
    throw new ReportError(
      "SPORE-REPORT-001",
      "Invalid runtime audit schemaVersion."
    )
  }
  return JSON.stringify(event) + "\n"
}
```

## validateAuditSafety()

```ts
export function validateAuditSafety(
  event: RuntimeAuditEvent
): CompilerDiagnostic[] {
  const diagnostics: CompilerDiagnostic[] = []

  if (event.schemaVersion !== "spore.runtime.audit.v1") {
    diagnostics.push({
      code: "SPORE-AUDIT-001",
      severity: "error",
      message: `Unsupported audit schema version: ${event.schemaVersion}.`
    })
  }

  if (!event.eventId || !event.executionId || !event.timestamp) {
    diagnostics.push({
      code: "SPORE-AUDIT-002",
      severity: "error",
      message: "Audit event is missing required identity or timestamp fields."
    })
  }

  if (containsUnsafeSecretValue(event.metadata)) {
    diagnostics.push({
      code: "SPORE-AUDIT-003",
      severity: "error",
      message: "Audit event metadata contains an unsafe secret value."
    })
  }

  if (event.status === "denied" && !event.denial) {
    diagnostics.push({
      code: "SPORE-DENIAL-001",
      severity: "error",
      message: "Denied audit event must include a DenialReport."
    })
  }

  if (event.proof && !hasFiveExecutionProofHashes(event.proof)) {
    diagnostics.push({
      code: "SPORE-PROOF-001",
      severity: "error",
      message: "Execution proof must include all five proof hashes."
    })
  }

  return diagnostics
}
```

## Diagnostic Codes

### SPORE-AUDIT

| Code | Meaning |
| --- | --- |
| `SPORE-AUDIT-001` | Unsupported audit schema version |
| `SPORE-AUDIT-002` | Missing required audit event field |
| `SPORE-AUDIT-003` | Unsafe secret value in audit metadata |
| `SPORE-AUDIT-004` | Invalid audit status transition |
| `SPORE-AUDIT-005` | Audit event ordering or chain failure |

### SPORE-REPORT

| Code | Meaning |
| --- | --- |
| `SPORE-REPORT-001` | Report schema version missing or unsupported |
| `SPORE-REPORT-002` | Report contains unsafe raw value |
| `SPORE-REPORT-003` | Report references missing artefact |
| `SPORE-REPORT-004` | Report hash mismatch |
| `SPORE-REPORT-005` | Report output path is unsafe |

### SPORE-PROOF

| Code | Meaning |
| --- | --- |
| `SPORE-PROOF-001` | Execution proof missing one or more of the five hashes |
| `SPORE-PROOF-002` | Proof hash format invalid |
| `SPORE-PROOF-003` | Manifest hash mismatch |
| `SPORE-PROOF-004` | Audit chain hash mismatch |
| `SPORE-PROOF-005` | Proof references missing execution id |

### SPORE-DENIAL

| Code | Meaning |
| --- | --- |
| `SPORE-DENIAL-001` | Denied event missing denial report |
| `SPORE-DENIAL-002` | Denial report missing evidence |
| `SPORE-DENIAL-003` | Secret boundary denial |
| `SPORE-DENIAL-004` | Policy denial |
| `SPORE-DENIAL-005` | Target compatibility denial |

### SPORE-EVIDENCE

| Code | Meaning |
| --- | --- |
| `SPORE-EVIDENCE-001` | Unknown evidence type |
| `SPORE-EVIDENCE-002` | Evidence missing required field |
| `SPORE-EVIDENCE-003` | Evidence contains unsafe raw secret |
| `SPORE-EVIDENCE-004` | Evidence hash mismatch |
| `SPORE-EVIDENCE-005` | Evidence references missing event or artefact |

## Testing Strategy

describe("validateAuditSafety", () => {
  it("rejects denied events without denial reports", () => {

      ...fixtureAuditEvent,

## Fail-Closed Rules

The audit/report layer must never:

```text
write raw secrets
write unsafe token values
write denial events without denial reports
write proof records without all five hashes
accept unknown evidence types silently
mark incomplete audit chains as verified
```

If audit safety validation fails, reports may be written only as failed safety reports and must not be marked verified.

## Summary

The v1 runtime audit model defines:

```text
RuntimeAuditStatus
RuntimeAuditEvent          (schemaVersion: "spore.runtime.audit.v1")
ExecutionProofHashes       (five confirmed fields: manifestSha256, auditSha256, evidenceSha256, denialSha256, artefactSha256)
ExecutionProofV1           (schemaVersion: "spore.execution.proof.v1")
ExecutionProofV2           (schemaVersion: "spore.execution.proof.v2" — extensible sections)
ExecutionProofSection
upgradeExecutionProofV1ToV2
DenialReport               (schemaVersion: "spore.denial.report.v1")
RuntimeEvidence
JsonlAuditWriter
serializeAuditEvent
validateAuditSafety
```

The canonical runtime audit format is JSONL.

The canonical execution proof contains five hashes — field names confirmed canonical:

```text
manifestSha256
auditSha256
evidenceSha256
denialSha256
artefactSha256
```

These contracts allow Galerina runtime, CLI, deploy, verify, explain, and audit tooling to produce safe, tamper-evident, machine-readable execution evidence.
