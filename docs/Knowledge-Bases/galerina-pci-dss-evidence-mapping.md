## The accuracy gap you MUST disclose to an assessor

`galerina-devtools-pci` (`pci-checker.ts`, `FUNGI-PCI-001..010`) models **7 of the 12** PCI requirement families.
**Unmodeled families are silently not-counted** — i.e. the checker can *over-report* compliance by omission (the
same unknown→pass fail-open class that bridge job 0084 generalises across OWASP/CWE/NIST/MITRE/SLSA). Under K3
this should resolve to an explicit **INDETERMINATE → fail-closed**, not a silent pass. Until that lands, any PCI
report generated by the tool must be accompanied by the 5/12 it does **not** model.

## What is genuinely defensible today (the honest pitch)

Galerina's real, enforced PCI value is **fail-closed admission + structural data-flow blocking + non-droppable
governance diagnostics** — compile-time + signed-manifest evidence that *replaces a slice of* manual code-review
sampling. It is **not** (yet) a substitute for CDE network isolation (that's aspirational #102–106) and its
signatures are **not** yet post-quantum (#34). Present the ✅/disclosed-◑ rows; never the ⚠ row as enforced.
