# LogicN — Governed Inference Tower

**Version:** 1.0 (2026-06-05)  
**Status:** Architecture specification — implementation roadmap active  
**Source:** Session analysis + vendor research (BitNet/NVFP4/GroqCloud)

---

## Overview

The Governed Inference Tower integrates three distinct inference engines as **first-class Tower Citizens**, governed by the unified `ai {}` contract block. Every inference call — regardless of hardware — produces a structured AuditEvent receipt and is constrained by the V_DPM capability register.

**Core principle:** We do not transpile or merge the engines. We govern their **entry, exit, and security boundary**.

---

## The Three-Tier Architecture

| Tier | Engine | License | Deployment | Role |
|---|---|---|---|---|
| **Tier 1** | BitNet b1.58 (Microsoft) | MIT | CPU, on-premise, air-gapped | Logic Auditor — deterministic, ternary weights |
| **Tier 2** | GroqCloud / NVIDIA LPX | MIT (LPU licensed) | Cloud API (WASI-HTTP) | Sequencer — latency-deterministic inference |
| **Tier 3** | NVFP4 (NVIDIA Blackwell) | Apache 2.0 | On-premise B200/RTX5090 | Executor — maximum throughput, bulk processing |

### Why this split

- **BitNet (Tier 1)** maps to LogicN's regulated market: healthcare, finance, government, defence. No cloud dependency. CPU-runnable on commodity hardware. Ternary weights {-1,0,+1} give bounded state space — closer to formal verification than any other LLM architecture.
- **GroqCloud/LPX (Tier 2)** serves cloud-native SaaS. WASI-HTTP wrapper works today with zero native code.
- **NVFP4 (Tier 3)** is the high-throughput path for organizations with Blackwell hardware. Apache 2.0 provides explicit patent grant — important for enterprise legal teams.

## The `ai {}` Contract Block

The developer declares intent and constraints. The `boot.lln` manifest maps governance tiers to hardware backends.

```lln
secure flow processDocument(doc: String) -> Result<Summary, Error>
contract {
  intent "Summarise a document using governed inference"
  effects { allow ai.inference, allow audit.write }
  ai {
    approved_models ["bitnet-b1.58-2b", "llama-3.3-70b-versatile"]
    
    ;; Fallback chain — if Tier 1 unavailable, try Tier 2
    fallback_approved ["groq-llama-3.1-8b"]
    fallback_requires network.outbound   ;; compiler checks this is declared
    
    ;; governance_tier drives engine selection — not precision_mode
    ;; "tier-1" → BitNet CPU   "tier-2" → GroqCloud   "tier-3" → NVFP4
    governance_tier "tier-1"
    
    ;; Audit trail — every call logs these fields to AuditEvent ledger
    audit_depth full   ;; token_count, latency_ms, input_hash, output_hash, model_version
    
    ;; Resource constraints — V_DPM enforces these as hard limits
    max_token_cost 500
    max_latency_ms 5000
    max_model_calls 1
  }
}
{
  trap doc == "" : ERR_EMPTY_DOCUMENT  ;; validated before inference call
  let summary = step ai::infer(doc)    ;; AuditEvent emitted by DSS.wasm
  return Ok(summary)
}
```

### `boot.lln` tier mapping

```lln
governance {
  ;; Map governance tiers to assimilated engine backends
  ;; The developer never changes flow code to switch engines — only boot.lln changes
  ai_tier_1: "logicn-ext-bridge-bitnet"   ;; BitNet CPU WASI-NN
  ai_tier_2: "logicn-ext-bridge-groq"     ;; GroqCloud HTTP wrapper
  ai_tier_3: "logicn-ext-bridge-nvfp4"    ;; NVFP4 TensorRT-LLM FFI
  
  assimilation_memory_budget: auto   ;; or explicit: 256MB
}
```

## The Promotion Pipeline CLI

**Core insight:** We don't transpile. We wrap, audit, and seal.

### Philosophy

- C++ stays as C++ — the math is already optimised by the hardware vendors
- LogicN governs the **boundary** — entry, exit, capability checks, audit trail
- The CLI generates the governance wrapper automatically from C++ headers

### CLI Commands

```bash
# Analyse a C++ function and generate the governed LogicN wrapper contract
logicn wrap ./bitnet_inference.cpp --name=BitNetInfer
# → Produces: bitnet_infer.lln (secure flow with ai {} contract)
#             bitnet_infer_host.rs (Wasmtime host function registration)

# Run full promotion pipeline: wrap + static analysis + seal
logicn promote ./bitnet-cpp --type=CPU_AUDITOR --tier=1
# → Produces: build/bitnet-cpu.wasm
#             build/bitnet-cpu.lmanifest (CBOR, ML-DSA-65 signed)

# Sign an existing WASM+manifest pair
logicn sign ./build/bitnet-cpu.wasm
# → Uses ML-DSA-65 keypair from governance/ directory
# → Updates .lmanifest with real GovernanceSignature (not placeholder)
```

### What `logicn wrap` generates

Given a C++ function:
```cpp
// bitnet_inference.cpp
InferenceResult bitnet_infer(const char* prompt, int max_tokens);
```

Generates:
```lln
// bitnet_infer.lln — auto-generated by logicn wrap
// DO NOT EDIT — regenerate from source with: logicn wrap ./bitnet_inference.cpp

import plugin assimilate "./bitnet-cpu.wasm" as BitNet {
  contract {
    intent "BitNet b1.58 2B — CPU-resident ternary inference"
    access {
      grant ai.inference   ;; V_DPM bit 5 pre-set at boot
      grant audit.write    ;; always permitted — AuditEvent emission
    }
  }
}

governed floor_2 flow bitnet_infer(prompt: String, maxTokens: Int) -> String
contract {
  intent "Invoke BitNet inference via governed WASI-NN host function"
  effects { allow ai.inference, allow audit.write }
  invariant {
    ensure prompt != ""
    ensure maxTokens > 0
    ensure maxTokens <= 4096
  }
}
{
  trap prompt == "" : ERR_EMPTY_PROMPT       ;; validated
  trap maxTokens <= 0 : ERR_ZERO_TOKENS     ;; validated
  trap maxTokens > 4096 : ERR_TOKEN_LIMIT   ;; ceiling enforced
  
  ;; step creates DWI, enforces V_DPM, emits AuditEvent on any fault
  let result = step BitNet.infer(prompt, maxTokens)
  return result
}
```

### The `.lmanifest` as Engine Passport

Every promoted engine gets an `.lmanifest` containing:

```json
{
  "engineId": "bitnet-cpu-v2b4t",
  "engineVersion": "2.0.0-4T",
  "sourceRepo": "https://github.com/microsoft/BitNet",
  "sourceCommit": "abc123...",
  "license": "MIT",
  "licenseFile": "LICENSE",
  "governanceTier": 1,
  "capabilities": ["ai.inference", "audit.write"],
  "assimilationMemoryMB": 450,
  "cborTag": 400,
  "governanceSignature": "ML-DSA-65:<hex>",
  "auditEventSchema": { "cborTag": 410, "version": "1.0" },
  "policyResolutionDag": { "cborTag": 416, "allowedEffects": 40 }
}
```

The Tower runtime refuses to load any engine without a valid, signed `.lmanifest`. This is the `CITIZEN_MISMATCH` trap.

## Wired Runtime: `logicn infer` (Brain → Brawn → Audit)

As of 2026-06-06 the Tower is wired into the LogicN runtime: a flow's `ai {}`
contract drives a **real, governed inference**, not just a compile-time check.

```bash
logicn infer examples/foundations/ai-inference-governed.lln \
       --prompt "Summarise the Q3 compliance report."
```

**Pipeline:**

1. **Read the contract.** The CLI parses the target flow's `ai {}` block from the
   AST — `approved_models`, `governance_tier`, `max_model_calls`, `max_token_cost`
   — and turns them into hard governance constraints.
2. **Brain.** `createHybridEngine()` (`logicn-tower-citizen`) plans the per-op
   precision (BitNet ternary / NVFP4 fp4 / fp8 / fp16) for the deployment tier.
3. **Brawn.** The engine's `BridgeRegistry` is supplied by
   `createCppBridgeRegistry()` (`logicn-ext-bridge-cpp`). Ternary ops dispatch to
   the **BitNet CPU bridge**; with no native addon compiled, the byte-faithful
   **TPL simulator** runs — `executedNatively: false` is reported honestly. The
   `ternaryChecksum` is bit-identical whether the stub or the cpp bridge runs
   (the determinism oracle, Citizen Standard 1).
4. **Hold-First governance.** An unapproved model or an exhausted call budget
   **traps before any compute runs** — `ERR_AI_MODEL_NOT_APPROVED` /
   `ERR_AI_CALL_BUDGET`, exit code 1, with only `LOAD → TRAP → ERASE` in the
   ledger. This makes `logicn infer` usable as a CI governance gate.
5. **Audit.** Every precision decision (with its bridge provenance) is written to
   an append-only JSONL ledger under `build/tower-logs/infer-<corr>.jsonl`.

**Seam remaining:** invoking inference from *inside* a flow body via
`AI.infer(...)` during `logicn run` needs the in-WASM `host:ai.infer` import
(string marshaling through the WAT host bridge). `logicn infer` is the Stage-A
host driver that delivers governed inference today; the in-WASM call is the
documented follow-up.
