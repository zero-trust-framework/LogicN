## Executive summary

The distinct issues cluster into four hotspots:

1. **WASM lowering correctness (`wat-emitter.ts`)** ‚Äî the Stage-B emitter silently
   mis-compiles or drops several live constructs: `#record-update` returns a null
   handle, Result `Ok/Err` and guard `when` arms fall through to interned-string
   equality, float arithmetic emits `i32` ops on `f64` operands, and a long tail of
   `(i32.const 0) ;; unhandled` placeholders export flows that silently compute 0.
   None emit a diagnostic. Real defects, but **gated behind the still-in-progress
   P9/Stage-B path** (the production Stage-A interpreter is correct), so mostly
   high/medium, not critical.

2. **Self-host parity blockers** ‚Äî `Array.count()` has no WAT lowering at all (used by
   every loop in all six self-hosted flows), and enum-variant `match tok.kind {‚Ä¶}`,
   Result `Ok/Err` arms, and a set of String methods
   (`slice`/`startsWith`/`endsWith`/`toLower`/`toUpper`/`trim`/`indexOf`/substring
   `contains`) are unmapped. These block lowering parser/type-checker/runtime to WASM
   parity.

3. **Host-fidelity / cross-runtime divergence (`wasm-runtime.ts`)** ‚Äî the WASM host
   uses UTF-16 indexing (`charCodeAt`/`.length`) where the interpreter uses code
   points, and overloads the value space with an in-band `-1` None sentinel that
   collides with legitimate `-1` results (`toInt`, `first`/`last`/`get`, `unwrapOr`).
   Latent today (live inputs are ASCII and non-negative); breaks the
   Stage-A==Stage-B==WASM parity contract for non-BMP or negative-int data.

4. **Security & governance** ‚Äî the **committed signing private key still in git
   history (#149)** is the single highest-impact item and remains critical/
   unremediated. The WASM admission gate's certified-profile claim is outside the
   signature pre-image, two CLI subcommands have shell-injection sinks, `keygen` writes
   the private key world-readable, and several fail-open footguns persist. In the Tower
   layer, the Epistemic-Hold 0‚Üí1 commit gate is dead/unwired, `signAudit()` forges an
   `mldsa65:` prefix over a keyless truncated hash, and the BitNet bridges define but
   never call `canCommit()`.
